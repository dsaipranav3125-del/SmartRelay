Problem Statement

Relay failures create avoidable downtime and cost.

Inverter relay failures are often detected only after service interruption. Existing setups lack early warning visibility and actionable failure forecasting.

Operational Issues

  • Unexpected relay breakdown causes unplanned downtime.
  • Maintenance teams react late due to limited predictive visibility.
  • Field service scheduling becomes urgent and inefficient.

Real-World Impact

  • Business interruption affects uptime commitments.
  • Emergency service and replacement raise maintenance cost.
  • Financial losses accumulate across multi-inverter fleets.

Proposed Solution

SmartRelay AI predicts failure before breakdown.

SmartRelay AI continuously evaluates relay health using telemetry and estimates Remaining Life and Failure Risk in real time.

Core Logic

  • Inputs: temperature, load %, switching cycles.
  • Predicts Remaining Life (%), Failure Risk (%), and likely failure horizon.
  • Enables maintenance action before system disruption.

Two-Layer Intelligence

  • Edge: immediate threshold alerts on device side.
  • Cloud: AI-driven predictive scoring and trend analysis.
  • Unified dashboard for operators and service teams.

System Architecture

End-to-end relay intelligence pipeline.

Inverter -> MODBUS -> IoT Device -> MQTT -> Cloud -> AI Model -> Dashboard

Inverter + MODBUS

Collects relay telemetry from device registers.

IoT Device

Packages and forwards data to cloud transport layer.

MQTT + Cloud

Streams telemetry at scale and stores it for inference.

AI + Dashboard

Computes risk/RUL and converts outputs to maintenance actions.

Model & Technical Approach

Lightweight, interpretable, and extensible model design.

Current prediction model

Linear regression-based scoring using live telemetry features. Designed for transparent logic and fast inference in demo and production-like environments.

Inputs

  • Temperature
  • Load %
  • Switching cycles

Outputs

  • Remaining Life (%)
  • Failure Risk (%)
  • Confidence Score

Future-ready extension

Architecture supports migration to advanced ML models such as Random Forest, Gradient Boosting, or sequence models without changing dashboard workflows.
